# 统计饼图放大与标签避让（Android）

**日期：** 2026-07-14  
**状态：** 设计已口头确认，待实现  
**端：** 仅 Android App（小程序本期不改）  
**关联：** `StatsScreen` / MPAndroidChart `PieChart`

## 背景

截图反馈：

1. 饼图上下空白过大，图本身偏小  
2. 小扇区外侧引线文字重叠（如卫浴 / 全屋智能 / 全屋净水）  

用户优先解「饼小 / 空白多」，且要求 **饼要大 + 文字也不挤**（方案 A）。

## 决策摘要

| 项 | 决定 |
|----|------|
| 做法 | 大饼 + 智能外侧标签（方案 A） |
| 范围 | 仅 Android |
| 图表库 | 继续用 MPAndroidChart，不换自绘 |

## §1 尺寸与空白

| 项 | 做法 |
|----|------|
| 图表高度 | 接近内容区宽度的正方形（`aspectRatio(1f)` 或 `height ≈ width`），去掉「容器很高但饼仍小」 |
| extraOffsets | 降到能放引线的最小值（约 8–12），避免空白垫高 |
| holeRadius | 略降（约 26–30），环面更厚 |
| 提示文案 | 保留一行小字，不占大块垂直空间 |

## §2 标签与避让

### 分层

| 占比 | 标注 |
|------|------|
| ≥ 12% | 扇区内：简称 + 百分比 |
| 约 3%–12% | 外侧引线；名称尽量 ≤3 字 + `%` |
| &lt; 约 3%，或同侧仍挤不下 | 图上不单独占行，**仅图例**（可点选高亮扇区） |

### 避让

1. 外侧标签按左 / 右分组  
2. 同侧按角度排序后做 **Y 错开**（最小间距约 14–16dp）  
3. 仍冲突 → 先缩短为只显示 `%`，再降级为仅图例  

### 图例

始终列出全部分组（含极小扇区）；点击高亮。小扇区信息以图例为主入口。

## §3 范围与验收

### 范围 / 非目标

- ✅ 仅 `StatsScreen` 饼图区布局与标签逻辑  
- ❌ 小程序、柱状图重做、自定义手绘饼、信息架构大改  

### 验收

1. 同数据下饼明显更大，卡片上下大块空白消失  
2. ≥12% 扇区内可读；3%–12% 外侧不大幅叠字；极小扇区可只在图例看到  
3. 图例完整且可点选高亮  
4. `sh oneClickSetup` 通过  

## 实现提示（非规范约束）

- MPAndroidChart 默认外侧标签不易完美避让时，可在 `ValueFormatter` / `PieDataSet` 参数上先收敛文案与阈值，必要时对过密切片在数据集层面合并展示策略（仍保留图例全量）  
- 避让若库能力不足：接受「降级到图例」路径，保证不叠字优先于「每片都出线」
