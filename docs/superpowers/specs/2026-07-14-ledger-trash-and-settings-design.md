# 账本垃圾箱 + 「我的」设置页设计

**日期：** 2026-07-14  
**状态：** 已批准，待写实现计划  
**端：** Android App（`/Users/beike/Projects/renovation-ledger`）+ 微信小程序（`/Users/beike/Projects/renovation-ledger-miniprogram`）  
**相关：** [多账本 Drawer](./2026-07-14-multi-ledger-drawer-design.md)（该文档原将「删除账本」列为非目标，**由本 spec 正式立项**）；[账本 autosave CSV](./2026-07-13-ledger-autosave-csv-design.md)

---

## 1. 背景与目标

多账本已支持新建 / 切换 / 重命名，尚无删除与恢复。用户需要：

1. **删除账本**前确认，并将完整 CSV 留在本地沙盒，可从「垃圾箱」恢复或永久删除。  
2. **「我的」右上角进入设置**：查看版本、改昵称与头像；业务入口（健康色、分类、导入导出、垃圾箱等）仍留在「我的」。

**成功标准：** 两端行为与文案一致（平台特有能力可差异实现，产品语义不单端遗漏）；删除 → 进垃圾箱（CSV + 索引）；可恢复；永久删除清文件；删当前/最后一本自动新建空账本并切换。

---

## 2. 决策摘要

| 项 | 选择 |
|----|------|
| 删最后一本活跃账本 | **允许** → 进垃圾箱，并**自动新建空账本**设为当前 |
| 删除入口 | **首页抽屉 +「我的」** 都能删（均需确认弹窗） |
| 实现路径 | **方案 C：库内硬删 + 仅靠沙盒 CSV 恢复**（不做软删字段） |
| 删除时 | 导出完整账本 CSV → 写/更新 trash 索引 → Room CASCADE / storage 硬删 project |
| 恢复时 | 从该 CSV 再导入成活跃账本 → 从索引移除 → 删 CSV |
| 永久删除时 | 再确认 → 从索引移除 + 删 CSV |
| 昵称 / 头像 | **挪进设置页**；「我的」不再内联编辑 |
| 健康色 | **仍在「我的」**，不进设置 |
| CSV 与 autosave | **不强制**与 `ledger_autosave.csv` 文件/格式字节级同一；trash 用**可完整还原**的导出格式 |

### 曾评估方案（已否决）

| 方案 | 说明 | 未选原因 |
|------|------|----------|
| A 软删 + 旁路 CSV | Project 加 `deletedAt`，活跃列表过滤 | 用户明确要求走 C |
| B 独立 Trash 表 | 删除时搬行再回灌 | 迁移/双写重，收益低 |

## 3. 数据与沙盒（方案 C）

### 3.1 活跃账本

- **Android：** Room `projects` / `budget_items` / `payments`（既有 CASCADE）；`current_project_id`（DataStore）指向活跃账本。  
- **小程序：** `renovation_ledger_v1` 内 `projects[]` + `items[]` + `currentProjectId`。  
- **不**在 Project 上增加 `deletedAt` 或软删标志。删除后库内 / storage 中不再保留该 project。

### 3.2 垃圾箱存储布局

**Android**

```text
context.filesDir/trash/
  index.json              # 垃圾箱条目列表
  {projectId}.csv         # 删除前导出的完整账本 CSV
```

**小程序**

```text
{USER_DATA_PATH}/trash/
  index.json
  {projectId}.csv
```

### 3.3 索引条目约定（`index.json`）

数组，每项至少包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 原 `projectId`（与 CSV 文件名对应） |
| `name` | string | 删除时账本显示名 |
| `deletedAt` | number | 删除时刻 epoch ms |
| `itemCount` | number | 删除时预算项数量（列表展示用；以导出前统计为准） |
| `csvPath` | string | 相对 `trash/` 的路径，固定为 `{id}.csv` |

排序：默认按 `deletedAt` **降序**（最近删除在前）。

### 3.4 Trash CSV 格式

- 必须能 **完整还原**：project 元数据 + 全部预算项 + 全部付款（与「完整导出 / round-trip」同能力）。  
- 实现上可复用 autosave v1 的序列化/反序列化，或其它已有完整导入路径；**不要求**与正在写的 `files/ledger_autosave.csv` 是同一文件，也**不要求**与手动分享用导出格式字节级一致。  
- 文件落在 `trash/{projectId}.csv`，与日常 autosave 路径隔离。

### 3.5 操作语义

| 操作 | 行为 |
|------|------|
| **删除（进垃圾箱）** | ① 对该账本导出完整 CSV → `trash/{id}.csv` ② 写入/更新 `index.json` ③ **硬删**库内该 project（Android CASCADE；小程序从 `projects`/`items` 剔除） ④ 若删的是当前账本：切到剩余任一活跃账本；若已无活跃账本：**新建空账本**并设为当前 |
| **恢复** | ① 读对应 CSV，导入为活跃账本（优先使用 CSV 内原 `project_id`，因其已硬删空出；若极端冲突则分配新 id 并仍视为恢复成功） ② 从 `index.json` 移除该条 ③ 删除该 CSV ④ **始终切换**到恢复后的账本为当前 |
| **永久删除** | 二次确认 → 从 `index.json` 移除 + 删除 CSV。不写回库 |

### 3.6 删最后一本 / 删当前本（细则）

- **删当前且仍有其它活跃本：** 硬删后把 `currentProjectId` 切到剩余列表中的一本（实现任选稳定策略，如列表第一本），并刷新 UI。  
- **删后零活跃本（含「最后一本」）：** 创建空账本（命名与现有「新建账本」一致，如「新账本」；不种默认模板，与 Drawer 新建约定对齐）→ 设为当前 → 保证首页始终有可显示账本。  
- **空账本同样可进垃圾箱：** 导出的 CSV 可几乎只有 project 行；恢复后仍为空账本。

### 3.7 失败与一致性

- **导出 CSV 失败：** 不写索引、不硬删；提示失败。  
- **已写 CSV + 索引后硬删失败：** 尽量回滚索引与 CSV（或提示「已备份但删除失败，请重试」）；不可出现「库已删但无 CSV」却声称进垃圾箱。优先保证：**要么仍在活跃库，要么 trash 中可恢复。**  
- **恢复时 CSV 缺失/损坏：** 提示失败，保留索引项（用户可再试或永久删除）。  
- **永久删除部分失败：** 尽量两边都清；列表**只展示 `index.json` 内条目**。无索引的孤儿 CSV 不展示、本期内不自动清扫（非目标：自动过期）；实现阶段不必为此加 UI。

## 4. UX

### 4.1 删除入口（两端一致）

1. **首页抽屉：** 每个账本提供删除（图标或侧滑/明确删除控件均可，需可发现）；点击后确认弹窗。  
2. **「我的」：** 账本列表或「管理账本」等价处可删；另有 **垃圾箱** 入口。  

确认弹窗文案要点（可微调，语义固定）：

- 将「{账本名}」移入垃圾箱  
- 会先导出备份，之后可从垃圾箱恢复  
- 永久删除前仍可找回  

确认后执行 §3.5「删除」。

### 4.2 「我的」页调整

| 保留 | 迁出 / 新增 |
|------|-------------|
| 健康色 | **新增** 垃圾箱入口 |
| 分类 / 标签（taxonomy）管理 | **新增** 右上角设置齿轮 |
| 导出 / 导入等业务入口 | **迁出** 本页内联昵称、头像编辑 → 设置页 |

Mine **不**再承担头像/昵称编辑；仅通过入口进入设置。

### 4.3 设置页（「我的」右上角进入）

| 能力 | 说明 |
|------|------|
| 版本 | Android `versionName`；小程序 `globalData.version`（或等价配置） |
| 昵称 | 可编辑并持久化（沿用现有 prefs / storage） |
| 头像 | 更换 / 清除；**小程序本期补齐头像能力**，与 Android 产品语义对齐 |
| **不含** | 健康色、垃圾箱内容管理、分类管理（那些留在「我的」或专用页） |

### 4.4 垃圾箱页

列表每项展示：

- 账本名  
- 删除时间（`deletedAt` 本地化展示）  
- 项数（`itemCount`）  

操作：

- **恢复** → §3.5；成功后提示，并切换到该账本为当前。  
- **永久删除** → 二次确认（文案强调不可恢复）→ §3.5。  

空垃圾箱：友好空态（无条目时说明「暂无已删除账本」）。

## 7. 验收标准

1. **删除：** 抽屉与「我的」均可删；确认后账本离开活跃列表，沙盒出现对应 CSV + `index.json` 条目。  
2. **删当前 / 最后一本：** 有其它本则切换；无活跃本则自动新建空账本并设为当前，首页不出现「无账本」死状态。  
3. **恢复：** 垃圾箱恢复后，项目/预算项/付款数据与删除前一致（在 CSV 完整前提下）；索引与 CSV 清掉。  
4. **永久删除：** 二次确认后索引条目与 CSV 文件均不存在；不可再恢复。  
5. **设置：** 可见版本；可改昵称与头像（含 MP 头像）；健康色仍在「我的」且功能可用。  
6. **交付：** Android `oneClickSetup` 通过；小程序同步实现且行为对齐。
