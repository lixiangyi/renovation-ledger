# 账单截图 / CSV 批量导入设计

**日期：** 2026-07-13  
**状态：** 待用户审阅（已根据旧 App CSV 导出补充）  
**关联：** `2026-07-13-renovation-ledger-design.md`（主产品设计）  
**样例数据：** `docs/superpowers/fixtures/dcjz_export_sample.csv`  
**背景：** 用户已有旧版「装修记账」App 账本截图与 CSV 导出，需迁入本 App，并沉淀为可复用的「多图识图 / 文件导入 → 批量确认 → 导入」能力。

---

## 1. 目标

1. **App 内**：支持一次选择多张账单截图，OCR 识别流水行，合并去重后经人工确认，批量导入为预算项。  
2. **本次迁移**：优先支持旧 App **原生 CSV 导出**（见 §7）；亦可整理 JSON；经确认页写入。  
3. **清单展示**：按大项分组（可折叠展开）；大项以旧账本类别为准（不止四类）。

成功标准：用 `dcjz_export_sample.csv`（66 行，合计约 ¥506,347.49）可完整导入；预计花费接近该合计；不自动产生虚假「已实付」（备注里的「已交」仅保留文案，见 §4.4）。

---

## 2. 已确认决策

| 决策点 | 结论 |
|--------|------|
| 截图类型 | 旧「装修记账」分类流水（日期 + 名称 + 金额），非单笔收据 |
| 落库形态 | 每行 → **待购买**预算项 |
| 金额字段 | `budgetAmount` = `contractAmount` = 导入金额（**分**；支持小数元如 12.7） |
| 大项映射 | 所属类别 → `stage`（见 §2.1 完整列表） |
| 付款 | **不**自动创建付款记录；状态保持待购买 |
| 顶栏预算/支出/剩余 | 不自动改写项目；确认页可展示对照提示 |
| 多图 | 一次多选；合并后按「名称 + 金额 + 日期」去重 |
| 产品形态 | 增强「图片识别」；**CSV 文件导入为迁移主通道** |
| 辅助通道 | **优先兼容旧 App CSV**；JSON 可选 |

```
路径 A：＋记一笔 → 图片识别 → 多选截图 → OCR → 合并去重 → 批量确认 → 导入
路径 B：我的 → 从文件导入 → 选 CSV/JSON → 解析 → 批量确认 → 导入
```

| 解析结果 | UI |
|----------|-----|
| ≥ 2 行 | 批量确认页 |
| 1 行 | 单笔 ConfirmEntry |

## 4. 数据模型

### 4.1 领域模型补充（相对主设计）

`BudgetItem` **新增**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `recordedDate` | `String?` | 记账日期 `YYYY-MM-DD`（CSV「记账日期」） |
| `remark` | `String` | 备注（CSV「备注」；默认 `""`） |

名称：CSV「建材名称」为空时，用「所属类别」作为 `name`（如全屋智能、全屋净水）。

金额：元（可小数）×100 **四舍五入**为分（`12.7`→`1270`，`399.64`→`39964`）。

### 4.2 中间态 `ImportedLineDraft`

```text
name, amountCents, recordedDate?, stage, remark,
sourceImageIndex?, isDuplicate, selected
```

去重键：`name.trim() + amountCents + (recordedDate ?: "")`。

### 4.3 导入写入

```text
BudgetItem(
  name, stage,
  budgetAmount = amountCents,
  contractAmount = amountCents,
  recordedDate, remark,
  payments = empty, isNewAddition = true
)
```

状态 → **TO_BUY**。

### 4.4 备注中的「已交」

样例：`已交2w`、`已交6000`。  
**不自动拆成付款**；备注全文保留。确认页提示用户稍后补付款。

## 5. UI

### 5.1 批量确认页

- 顶栏：已选 N/M；副文案：来源（X 张图 / 文件名）、去重 Y 条、合计金额  
- 提示：待购买；预算=合同价；备注含「已交」不自动记已付  
- 按 `stage` 分组；组头：项数 + 合计  
- 行：勾选、日期、名称、金额；有备注显示一行灰字；重复标「重复」  
- 可编辑：名称 / 金额 / 大项 / 备注 / 日期  
- 全选、仅选非重复、确认导入  
- UI 遵循主设计 §8.0（标题栏、Card、Material 3）

### 5.2 清单分组展开

家具 / 家电 / 软装 / 硬装 / 卫浴 / 全屋定制 / 全屋智能 / 全屋净水 …  
默认折叠；展开见小项；详情可编辑 `remark`、`recordedDate`。

### 5.3 入口

图片识别 + **我的 → 从文件导入**（迁移优先）。

## 8. 错误处理

表头不匹配提示用旧 App 导出格式；金额非法标红；小数必须单测覆盖。

- 空名称 → name=stage
