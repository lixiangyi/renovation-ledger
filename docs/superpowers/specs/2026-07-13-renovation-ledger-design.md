# 装修记账（Renovation Ledger）产品设计

**日期：** 2026-07-13  
**状态：** 待用户审阅  
**产品定位：** 面向夫妻/家人协作的装修预算与付款追踪工具，解决「已花多少 / 还要花多少 / 超支多少」看不清的问题。

---

## 1. 背景与痛点

市面装修记账 / 通用记账 App 普遍存在：

1. **总预算与分项预算脱节**：不能「分项相加 = 总预算」，或两种填法不能并存。
2. **付款状态粗粒度**：定金已付、尾款已付/未付、未定商家（仅预算）混在一起，缺少清晰状态机。
3. **总览口径混乱**：无法同时回答「现在花了多少」「还有多少尾款要付」「还有多少东西没买」「买完预计总共多少、超支多少」。

本产品以**财务模型 + 总览展示**为差异化核心，不做装修公司对接、不做优惠导购。

---

## 2. 目标用户与协作

| 项 | 决策 |
|----|------|
| 用户 | 夫妻 / 家人共同管一套房的装修账（2–3 人） |
| 权限 | 成员平等，均可增删改预算项与付款 |
| 同步 | 实时同步；每条记录显示「谁记的」 |
| 冲突 | 同一项同时编辑 → 后保存覆盖，并提示「对方刚改过」 |
| 邀请 | 邀请码 / 微信分享加入同一项目 |
| 退出 | 可自行退出；发起人可移除成员；项目至少保留 1 人 |

## 3. 平台路线

```
Phase 1  Android 国内版（优先：开发者熟悉，离线记账 + 本地调试方便）
Phase 2  微信小程序（获客 / 伴侣零安装协作，后续再做）
Phase 3  iOS
```

- 各端共用同一套指标口径；后端与数据互通在协作同步阶段引入。
- Android 国内版优先支持**离线记账，有网后自动同步**。
- 不依赖 Google 服务（无 Firebase / Play Services 强依赖）；同步与语音/OCR 选用国内可用方案。
- 实现细节见 `docs/superpowers/plans/` 下的实现计划。

## 4. MVP 范围

### 必做

- 分项预算汇总为总预算（可只填总预算，也可分项相加）
- 预算项三态：待购买 / 付款中 / 已结清
- 定金 / 尾款（及其他）付款记录；已付 / 未付
- 首页总览：已实付、待花费（含待付尾款 + 待购买）、预计花费与超支
- 待花费展开与二级明细页
- 双人/三人账号协作 + 实时同步
- 内置装修预算模板
- 按阶段 / 分类 / 空间统计（饼图 + 柱状图 + 列表）
- 预算健康色（绿 / 橙 / 红，可关闭）
- **三种录入方式：手动录入、语音录入、图片识别录入**
- 付款凭证拍照（可与图片识别共用拍照入口）
- 导出 Excel

### 明确不做（MVP）

- 装修公司 / 平台对接
- 省钱特惠频道、社交分享装修成果
- 多工地重度管理（装修公司多角色）
- 系统推送付款提醒（可后续版本）
- 施工日记、工期甘特图（可后续版本）
- 语音/识图结果**自动落账不经确认**（必须人工确认后再保存）

## 5. 核心概念与状态机

### 5.1 实体

```
装修项目 (Project)
  └── 预算项 (BudgetItem)
        ├── 名称、阶段(stage)、分类、空间（可选）
        ├── 预算金额 (budgetAmount)
        ├── 合同总价 (contractAmount，可选)
        ├── 记账日期 (recordedDate，可选，YYYY-MM-DD)
        ├── 备注 (remark)
        ├── 商家信息（可选）
        ├── 是否中途新增（标记）
        └── 付款记录 (Payment) × N
              ├── 类型：定金 | 尾款 | 其他
              ├── 金额
              ├── 状态：已付 | 未付
              ├── 日期、备注
              └── 凭证照片
```

阶段 `stage` 常用值（导入旧账本时完整支持）：家具、家电、软装、硬装、卫浴、全屋定制、全屋智能、全屋净水；亦可自定义。

### 5.2 预算项状态

| 状态 | 条件 | 计入指标 |
|------|------|----------|
| **待购买** | 无任何付款记录（含：仅预算；已填合同价但零付款） | → 待购买 |
| **付款中** | 有付款且未结清 | → 已实付 + 待付尾款 |
| **已结清** | 全部付清（自动判断或手动「标记已结清」） | → 仅已实付 |

**effectiveCost（单项预计花费）**

- 已填写合同总价 → 用合同总价
- 未填写合同总价 → 用预算金额

已签约但零付款 → 仍为**待购买**，effectiveCost 用合同价。

### 5.3 状态流转

```
待购买 ──首次「已付」付款──→ 付款中 ──全部付清 / 标记已结清──→ 已结清
```

待购买阶段可随时：改预算、填商家、填合同价（填完仍为待购买，直到第一次已付）。

## 6. 财务指标定义

统一口径（全端一致）：

| 指标 | 公式 |
|------|------|
| **总预算** | 各预算项预算金额之和（也可先填总额再拆分，拆分后以分项之和为准） |
| **已实付** | Σ 所有状态为「已付」的付款金额 |
| **待付尾款** | Σ 「付款中」项目下状态为「未付」的付款金额（通常为尾款，也可含其他未付） |
| **待购买** | Σ 所有「待购买」项的 effectiveCost |
| **待花费** | 待付尾款 + 待购买 |
| **当前超支** | 已实付 − 总预算（正数=超支，负数=节余） |
| **预计总花费** | Σ 所有当前存在预算项的 effectiveCost（**含中途新增项**） |
| **预计超支** | 预计总花费 − 总预算 |

说明：

- 「剩余预算 = 总预算 − 已实付 − 待付尾款」不作为首页主指标；首页以「已实付 / 待花费」为主。
- 预计总花费包含装修过程中新增的项目，反映真实最终花费预期。

### 示例（总预算 40 万）

| 指标 | 数值 |
|------|------|
| 总预算 | 40w |
| 已实付 | 43w → 当前超支 +3w |
| 待付尾款 | 5w |
| 待购买 | 3w |
| 待花费 | 8w |
| 预计总花费 | 51w → 预计超支 +11w |

## 8. 页面设计

### 8.0 UI 总则（原质感 / Material 3）

| 规则 | 说明 |
|------|------|
| **标题栏** | **所有页面**（含四 Tab 根页与二级页）必须有 `TopAppBar` 标题栏；根页显示页面名（总览/预算清单/统计/我的），二级页带返回 + 标题 |
| **设计语言** | **Material 3 原质感**：圆角、色调表面（`surface` / `surfaceContainer`）、系统/Material Icons，避免自定义花哨阴影堆叠 |
| **模块容器** | 页面内独立信息块优先用 **Card**（如总览「已实付」「待花费」「预计花费」、统计图表区、我的设置项） |
| **图标** | 核心指标与入口配图标（已实付、待花费、预计、记一笔等）；图标随健康色或 `onSurfaceVariant` 着色 |
| **待花费结构** | 首页展开区与「待花费明细」页统一为 **双 Tab**：`待付尾款` \| `待购买`（不再用上下两块长列表并排堆叠） |

主题：以 Material 3 浅色方案为默认；健康色规则仍见 §9（可关）。

### 8.1 总览（首页）

**标题栏：** `总览`（可副标题显示项目名，或标题下小字「我家装修 · 成员」）。

**总预算：** 大字居中，可放在 Card 内。

**两列核心指标（各为 Card，带图标）：**

| 列 | 图标建议 | 内容 |
|----|----------|------|
| 已实付 | 付款/钱包类 | 金额；下方「超支 +X」或「节余 X」 |
| 待花费 | 待办/清单类 | 合计；列内摘要两行（待付尾款 / 待购买金额）；点击展开 |

**待花费展开区（双 Tab）：**

- Tab1 **待付尾款** / Tab2 **待购买**
- 当前 Tab 下列表，**最多 5 条**；超过显示「…还有 N 项」
- 底部：「查看全部待花费明细 ›」→ 二级页（默认打开与当前 Tab 相同的 Tab）
- 行点击 → 预算项详情

**预计花费 Card**（带趋势/预警图标）+ **进度条**（可同放一 Card 或分两 Card）。

**最近记账**列表 + FAB「＋记一笔」。

### 8.2 待花费明细（二级页）

- **标题栏：** 返回 +「待花费明细」；可显示合计
- **双 Tab：** `待付尾款` \| `待购买`（与首页展开一致）
- 当前 Tab 下**完整列表不截断**；列表项可用 Card 或标准 ListItem
- 行 → 预算项详情

### 8.3 预算清单

- **标题栏：**「预算清单」+ 右侧「＋新增」
- 状态筛选 Chip：全部 / 待购买 / 付款中 / 已结清
- 按 **stage** 分组，**默认可折叠**；组头合计；组内项用 Card
- 中途新增可角标「新增」

### 8.4 预算项详情

- **标题栏：** 返回 + 项名称
- 信息区、付款列表用 Card 分区
- 操作：添加付款、标记已结清、编辑、删除

**添加付款弹层字段：** 类型（定金/尾款/其他）、金额、状态（已付/未付）、日期、备注、凭证（拍照/相册）。

### 8.5 统计

- **标题栏：**「统计」
- 维度 / 饼图数据源切换；图表与 TOP5 分区用 Card

### 8.6 我的

- **标题栏：**「我的」
- 成员、健康色开关、导出、从文件导入、设置等分 Card 展示

### 8.7 录入方式（记一笔 / 添加付款 / 新建预算项）

点击「＋记一笔」先选择录入方式，再进入对应流程。三种方式最终都落到**同一套确认表单**（可改字段后再保存），避免误识别直接入账。
录入相关页（手动/确认/批量导入确认）均须有标题栏。

```
＋记一笔
├── 手动录入
├── 语音录入
└── 图片识别
        ↓
   确认表单（名称/金额/类型/关联预算项/已付未付…）
        ↓
   保存 → 写入预算项或付款记录
```

#### A. 手动录入

- 传统表单：选已有预算项或新建项 → 填金额、类型（定金/尾款/其他）、已付/未付、日期、备注、凭证。
- 也可从预算清单「＋新增」直接新建预算项（仅预算、待购买）。

#### B. 语音录入

- 用户说出自然语言，例如：「瓷砖定金三千五已付」「灯具预算六千还没买」。
- 系统解析出候选字段：关联项名称、金额、付款类型、已付/未付、是否新建项等。
- **必须进入确认页**，用户核对/修改后再保存。
- 识别失败：提示重说或改用手动。
- 小程序：优先用微信同声转写 / 官方语音能力；Android：系统语音或国内 ASR 服务（实现计划再定）。
- 离线：Android 离线场景下语音不可用时，引导改用手动；有网后再用语音。

#### C. 图片识别录入

- 拍照或从相册选图：收据、报价单、合同金额页、转账截图等。
- OCR / 识图提取：金额、日期、商家/品名（能识别则填）；原图同时可作为**凭证**挂到该笔付款。
- 同样进入确认页，用户核对后保存。
- 识别不到关键字段（如金额）时，预填已识别部分，其余手动补全。
- 与「付款凭证」关系：识图用的照片默认保存为凭证；用户也可只当凭证上传、不做识别。
- **账单截图批量导入**（旧账本多行流水、多图合并去重、清单大项分组展开）：见专项设计 `2026-07-13-screenshot-import-design.md`。

#### 确认页统一字段（三种入口共用）

| 字段 | 说明 |
|------|------|
| 动作类型 | 记一笔付款 / 新建预算项 / 两者都有（如语音「买了灯具花了六千」） |
| 关联预算项 | 匹配已有项，或新建 |
| 金额、付款类型、已付/未付、日期 | 付款场景 |
| 预算金额 / 合同价 | 新建或更新项时 |
| 来源标记 | 手动 / 语音 / 识图（便于排查） |
| 原始素材 | 语音文本或图片（可选保留） |

## 9. 预算健康色（主题色）

用三档颜色表达「预算是否健康」，作为产品主题强调色（首页数字、进度条、预计卡片、统计超支等）。用户可在「我的」关闭此功能。

### 9.1 分档规则

以 **预计超支率** 为主题色依据：

```
预计超支率 = 预计超支 / 总预算
           = (预计总花费 − 总预算) / 总预算
```

| 档位 | 条件 | 颜色 | 含义 |
|------|------|------|------|
| 预算内 | 预计超支率 ≤ 0 | **绿色** | 预计花费未超过总预算 |
| 轻度超支 | 0 < 预计超支率 ≤ **用户设定阈值**（默认 15%，可调 1%～100%） | **橙色** | 超支，但在阈值以内 |
| 重度超支 | 预计超支率 > 用户设定阈值 | **红色** | 超支超过阈值 |

总预算为 0 时：不染色，使用中性色。

### 9.2 作用范围（功能开启时）

| 位置 | 行为 |
|------|------|
| 首页「已实付」超支/节余文案 | 按**当前超支率**（已实付−总预算）/总预算 套用同档绿/橙/红 |
| 首页「预计花费」卡片边框与「预计超支」文案 | 按**预计超支率**套用主题色 |
| 首页双进度条填充色 | 按对应口径（预算执行用当前超支率，预计执行用预计超支率） |
| 清单 / 详情单项「合同价超预算」 | 单项超预算率 = (合同价−预算)/预算；同档绿/橙/红（≤0 绿） |
| 统计页阶段/分类预计超支、TOP5 | 按该组或该项超预算率套用同档颜色 |
| 全局页面背景、模块表面色、底部 Tab | 按**预计超支率**切换整套 Material 主题（绿/橙/红系） |

### 9.3 关闭行为

- 设置项名称建议：**预算健康色**；另提供 **轻度超支上限** 滑条（1%～100%，默认 15%）
- 默认：**开启**
- 关闭后：上述位置一律使用中性色（如主文字色 / 灰色进度条），仍展示「超支 +X」等数字文案，只是不再绿橙红染色；全局背景与 Tab 也回到中性主题
- 协作项目：该开关与阈值为**个人偏好**（每人自己关/调，不影响对方）

### 9.4 与旧预警规则的关系

原「超 10% 加强提示」「单项超 20% 标红」统一收敛为本节三档规则（默认阈值 15%，可调），避免多套阈值并存。

MVP 不做系统推送通知。

## 11. 导出

- 支持将预算项、付款记录导出为 Excel，便于对账与留存
- 字段至少包含：项名称、阶段/分类、预算、合同价、状态、各笔付款（类型/金额/已付未付/日期）、记账人

---

## 12. 成功标准（MVP）

用户打开首页，无需计算即可回答：

1. 现在已经实付了多少？是否已超初始预算？
2. 还要花多少（待花费）？其中多少是尾款、多少是还没买的？
3. 如果现在清单上的东西全部买完/付完，预计总共多少、超支多少？
4. 钱主要花在哪些阶段/分类？哪几项涨价最多？
5. 另一半刚记的一笔，自己这边能马上看到。

## 14. 已确认决策摘要

| 决策点 | 结论 |
|--------|------|
| 协作模式 | 夫妻/家人，平等记账，实时同步 |
| 离线（Android） | 支持离线记账，有网自动同步 |
| 预计花费范围 | 含中途新增项 |
| 已签约零付款归属 | 待购买（不是待付尾款） |
