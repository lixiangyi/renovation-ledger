# 装修记账 · Renovation Ledger

面向夫妻 / 家人的**装修预算 + 付款追踪**工具。  
一眼看清：**已实付多少、还要花多少、预计总共多少、超了还是省了。**

当前版本 **v0.1.0**（本地优先 MVP）。  
本仓库为 **Android**；微信小程序在本地姊妹目录 `renovation-ledger-miniprogram`（产品能力与本 README「功能一览」对齐，细节以该仓 README 为准）。

---

## 为什么做这个

通用记账和部分装修 App 常见痛点：

1. **总预算与分项对不上** — 不知道「分项加起来」到底是不是总预算  
2. **付款状态糊在一起** — 定金、尾款、还没买的东西混为一谈  
3. **总览口径混乱** — 回答不了「现在花了多少 / 还要付多少 / 买完会不会超」

本产品以**财务模型 + 总览指标**为核心差异，不做装修公司对接、不做优惠导购。

## 核心概念（30 秒）

| 概念 | 说明 |
|------|------|
| **账本** | 一套房 / 一次装修对应一个账本；支持多账本切换 |
| **预算项** | 一条分项预算（如「全屋定制」），可挂多笔付款 |
| **三态** | **待购买** → **付款中** → **已结清** |
| **付款** | 已付 / 未付；支持定金、尾款等 |
| **总预算** | 各预算项 `预算金额` 之和 |
| **已实付** | 所有「已付」付款之和 |
| **待花费** | 待付尾款 + 待购买金额 |
| **预计花费** | 按当前合同/预算口径推算的买完总花费 |
| **超支 / 节余** | 相对总预算：当前（已实付）与预计两套口径；单项也可看「已付 vs 预算」 |

## 功能一览（已上线）

### 四个主 Tab

#### 1. 总览

- **总预算**大数字、健康色整页主题（绿 / 橙 / 红，可关）
- **已实付**：展开后见「超支项 / 节余项」Tab；合计在 Tab 下方；可进「已实付明细」
- **待花费**：待付尾款 + 待购买；可进二级明细
- **预计花费**与执行进度（预算执行 / 预计执行）
- 右上角汉堡菜单：**多账本抽屉**（切换 / 新建 / 重命名）
- FAB **记一笔**

#### 2. 清单

- 按**阶段**分组的预算项列表  
- 状态筛选（待购买 / 付款中 / 已结清）  
- 进入详情增改付款、改预算项

#### 3. 统计

- 按 **分类 / 阶段 / 空间** 聚合  
- 饼图指标可切：**已实付 / 预计花费 / 预算**  
- Android 另有柱状对比与「合同超预算 TOP5」  
- 小程序为 Canvas 饼图 + 图例列表

#### 4. 我的

- 昵称、头像（Android）；昵称 / 账本名称（小程序）  
- **预算健康色**开关 +「轻度超支」橙色上限比例（默认约 15%）  
- **标签管理**：自定义阶段 / 分类 / 空间  
- **导出 CSV**、**导入旧账本 CSV**（导入会**新建账本**再写入，避免污染当前账）  
- 小程序额外支持「重置示例数据」

### 记一笔与详情

| 能力 | 说明 |
|------|------|
| 手动记一笔 | 新建预算项，或给已有项记一笔付款（阶段 / 分类 / 空间） |
| 详情页 | 付款列表、补记付款、标记结清、编辑 / 删除预算项 |
| 待花费明细 | 「待付尾款」「待购买」两个 Tab |
| 已实付明细 | 「超支项」「节余项」；超支/节余合计 |

> Android 录入选择器里预留了「语音 / 拍照」入口，当前为**演示草稿确认流程**，尚未接入真实语音识别与 OCR。小程序本期不做语音 / 识图。

### 多账本

- 总览侧滑抽屉：**切换 / 新建 / 重命名**  
- CSV **导入**时提示并将数据写入**新账本**后切换过去（当前账本保留）  
- 各账本数据隔离

### 数据耐久（Android）

- 变更后自动写入本地 autosave CSV（应用私有目录 + Downloads 尽力备份）  
- 若库空但检测到备份，启动时询问是否恢复  

小程序侧重手动导出，不做自动备份。

## 预算健康色

跟随**预计超支相对总预算**的比例：

| 状态 | 含义（默认可调） |
|------|------------------|
| 🟢 绿 | 预算内或未超支 |
| 🟠 橙 | 轻度超支（默认不超过预算 15%） |
| 🔴 红 | 重度超支（超过上述比例） |

可在「我的」关闭整页换色；关闭后列表里的超支/节余金额仍可用红/绿提示。

## 快速开始（Android）

**环境：** JDK 17、Android SDK、已连接设备或模拟器（`adb devices` 可见）

```bash
cd /path/to/renovation-ledger
sh oneClickSetup
```

脚本会：`gradlew clean` → `assembleDebug` → `adb install -r` → 启动 `MainActivity`。

也可以用 Android Studio 打开本仓库，选 `app` 模块直接 Run。

### 小程序

1. 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)  
2. 导入姊妹仓目录  
3. AppID 可用测试号 / `touristappid`  
4. 编译预览

## 设计原则（摘录）

- **先确认再落账**：语音 / 识图（未来）不得自动静默入账  
- **口径可解释**：总览数字能追溯到预算项与付款状态  
- **本地可用**：Android 离线记账优先；同期双端产品语义对齐  
- **导入不污染**：CSV 导入默认新账本
